Vi skall skapa en ny användare i databasen, en som har fulla behörigheter och vi kallar den "dbadm".



## Starta terminalklienten som root

Börja med att starta terminalklienten för MariaDB som root-användaren.

```bash
sudo mariadb
```

Nu är du redo att skriva SQL-satser för att skapa användaren.



## Skapa användaren "dbadm"

Kör följande SQL-sats, i terminalklienten, för att skapa din användare och ge den tillåtelse att koppla sig mot databasservern oavsett vilken maskin klienten finns på `@'%'`.

```sql
CREATE USER 'dbadm'@'%'
IDENTIFIED BY 'P@ssw0rd';
```

Kör följande sats för att ge användaren fullständiga rättigheter på alla databaser och alla tabeller `*.*`, det blir i princip samma rättigheter som en root-användare som har tillgång till allt.

```sql
GRANT ALL PRIVILEGES
ON *.* TO 'dbadm'@'%'
WITH GRANT OPTION;
```

Om något går fel kan du ta bort användaren med kommandot DROP och sedan börja om igen.

```sql
DROP USER IF EXISTS 'dbadm'@'%';
```

Prova nu om du kan logga in med din nya användare. Avsluta din nuvarande databaskoppling (skriv "exit") och öppna en ny så här.

```sql
mariadb -udbadm -pP@ssw0rd
```

Prova även att öppna den så här genom att skriva in lösenordet direkt i terminalen.

```sql
mariadb -udbadm -p
```

Du kan kontrollera att du nu är inloggad som din "dbadm@%" användare.

```text
MariaDB [(none)]> SELECT CURRENT_USER();
+-----------------+
| USER()          |
+-----------------+
| dbadm@%         |
+-----------------+
```

Du kan också kontrollera vilka rättigheter din användare har med kommandot `SHOW GRANTS;`.

```sql
SHOW GRANTS FOR CURRENT_USER();
```

Tills vidare skall du nu använda din "dbadm" användare istället för root användaren.



## Avslutningsvis

Du har nu skapat en användare "dbadm" i databasen MariaDB som du kan använda för att skapa databaser och tabeller med fullständiga rättigheter.
