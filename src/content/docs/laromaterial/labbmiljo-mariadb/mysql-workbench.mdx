Vi installerar desktopklienten MySQL Workbench och kopplar upp den mot en lokal databasserver i form av MariaDB. MySQL Workbench är en populär klient som fungerar bra mot MariaDB och MySQL databaser.



## Förutsättning

Du har installerat databasservern MariaDB och du kan koppla upp dig mot den via terminalklienten.

Du har skapat en lokal användare `dbadm` i databasservern.



## Ladda hem och installera MySQL WorkBench


[MySQL Workbench](https://dev.mysql.com/doc/workbench/en/) är en desktopklient du kan använda för att jobba mot en databasserver i form av MySQL eller MariaDB. En desktopklient är ett bra alternativ och komplement till den terminalbaserade klienten.

Följ instruktionerna på sidan och ladda hem och installera rätt version som matchar ditt system.

Det är relativt smidigt att installera via installationsprogram till Windows och macOS. Det kan vara lite pilligare att installera Linux-varianten, sök hjälp på nätet om det krånglar.

När installationen är klar, starta applikationen MySQL Workbench.

## Skapa koppling till databasservern

Skapa en ny "MySQL Connection" och använd din användare "dbadm" (som du skapade tidigare) för att logga in till databasservern.

Det kan se ut så här när du konfigurerar din uppkoppling.

import wb2 from '@assets/databas/installera-wb/wb2.png';

<Figure 
    src={wb2}
    caption="Fyll i detaljer för din uppkoppling."
/>

Det du behöver ändra är följande:

* Connection Name: Ge din uppkoppling ett namn, tex "dbadm".
* Username: Skriv in användaren "dbadm".
* Password: Klicka på "Store in Vault/KeyChain..." för att skriva in ditt lösenord för användaren "dbadm".

Pröva med knappen "Test Connection" för att se om uppkopplingen lyckas. Workbench försöker då koppla upp sig mot din databasserver med de detaljer du angett.

Det kan hända att du får en varning då workbench är utvecklad främst mot databasen MySQL, men eftersom MariaDB är en fork av MySQL så fungerar det ändå.

Spara uppkopplingen när den fungerar.



## Problem med SSL?

Om du får problem med krav på SSL uppkoppling så kan du lösa det genom att gå till fliken "Advanced" och i fältet "Others" skriver du in `useSSL=0`.

## Problem att logga in?

Om du får problem att logga in och användaren eller lösenordet inte stämmer, då kan du dubbelkolla att du verkligen kan logga in med terminalklienten med din användare dbadm.

Öppna en terminal.

```text
mariadb -udbadm -p
```

Du bör kunna köra ovan kommando, skriva in ditt lösenord, och sedan logga in med terminalklienten. När det fungerar så vet du vilket lösenord du kan använda tillsammans med användaren dbadm. Det bör även fungera i workbench.



## Testa MySQL WorkBench

Använd din koppling till databasen. Dubbelklicka på kopplingen och workbench kopplar sig mot din databashanterare och du får upp en ny flik för uppkopplingen.

import wb4 from '@assets/databas/installera-wb/wb4.png';

<Figure 
    src={wb4}
    caption="Nu är du uppkopplad mot databasservern och kan skriva kommandon."
/>

Du kan nu skriva SQL-kommandon i fönstret. Testa att köra några av följande kommandon för att se att allt fungerar.

```sql
SELECT USER();
SELECT CURRENT_USER();

SELECT VERSION();

SELECT NOW() AS 'The local time is';

SHOW DATABASES;
```

I workbench kan du placera markören på en rad och trycka `ctrl-enter` för att enbart köra det kommandot. Använd `cmd` istället för `ctrl` på macOS. Vill du köra alla kommandon i fönstret så trycker du `shift-cmd-enter` (macOS) eller `shift-ctrl-enter` (Windows/Linux).
